**src/core/types.ts:**

Define branded types using `unique symbol` for compile-time separation:

```typescript
declare const _gsTag: unique symbol
declare const _pvTag: unique symbol

export type GameState<S> = S & { readonly [_gsTag]: never }
export type PlayerView<S> = S & { readonly [_pvTag]: never }

export function makeGameState<S>(state: S): GameState<S> {
  return state as GameState<S>
}

export function makePlayerView<S>(view: S): PlayerView<S> {
  return view as PlayerView<S>
}
```

Note: Both use `S &` (not `Partial<S>`) — the game layer decides what fields PlayerView contains. The brand tag is what creates the compile-time incompatibility. The `S` type parameter for PlayerView will typically be a different (narrower) type than the `S` used for GameState.

Define GameConfig Zod schema:
```typescript
import { z } from 'zod'

export const PlayerConfigSchema = z.object({
  id: z.string(),
  name: z.string(),
  model: z.string().optional(),
  persona: z.string().optional(),
})

export type PlayerConfig = z.infer<typeof PlayerConfigSchema>

export const GameConfigSchema = z.object({
  gameId: z.string(),
  seed: z.number().int(),
  players: z.array(PlayerConfigSchema).min(1),
  roles: z.array(z.string()).optional(),
})

export type GameConfig = z.infer<typeof GameConfigSchema>
```

Define GameOutcome:
```typescript
export const GameOutcomeSchema = z.object({
  winner: z.string(),
  playerRoles: z.record(z.string(), z.string()),
  metadata: z.record(z.string(), z.unknown()).optional(),
})

export type GameOutcome = z.infer<typeof GameOutcomeSchema>
```

**src/core/game.ts:**

```typescript
import type { GameState, PlayerView, GameConfig, GameOutcome } from './types.js'

export interface Game<S, A> {
  initialState(config: GameConfig): GameState<S>
  getPlayerView(state: GameState<S>, playerId: string): PlayerView<S>
  getValidActions(state: GameState<S>, playerId: string): A[]
  applyAction(state: GameState<S>, playerId: string, action: A): GameState<S>
  isTerminal(state: GameState<S>): boolean
  getOutcome(state: GameState<S>): GameOutcome | null
}
```

Note: `getPlayerView` accepts `GameState<S>` and returns `PlayerView<S>` — this is the information boundary. The `S` type parameter represents the raw state shape; the branded wrappers ensure callers can't pass a PlayerView where a GameState is expected, or vice versa.

**src/core/player.ts:**

```typescript
import type { PlayerView } from './types.js'

export interface Player<S, A> {
  readonly id: string

  act(view: PlayerView<S>, validActions: A[]): Promise<A>
}
```

Note: `act()` takes `PlayerView<S>`, NOT `GameState<S>`. This is the compile-time information hiding boundary per FRAME-01 and CONTEXT.md locked decisions.
